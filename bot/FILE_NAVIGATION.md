# Навигация по проекту

Этот файл нужен как карта проекта: куда идти, если нужно менять конкретную часть бота.

## Запуск

- `src/main.py` — главная точка входа. Здесь создаются таблицы, регистрируются команды Telegram и запускается polling.
- `Procfile` — команда запуска для Railway: `worker: python -m src.main`.
- `requirements.txt` — Python-зависимости проекта.

## Настройки

- `src/config.py` — все переменные окружения: `BOT_TOKEN`, `ADMIN_CHAT_ID`, `ADMIN_IDS`, `DATABASE_URL`, `ADMIN_USERNAME`.
- `.env.example` — пример локального `.env`.
- `docker-compose.yml` — локальный PostgreSQL для тестов.

## База данных

- `src/database.py` — SQLAlchemy async engine, async-сессии, создание таблиц и простые автоколонки.
- `src/models/order.py` — модель таблицы `orders` и статусы заявки.
- `src/repositories/order_repository.py` — SQL-запросы к заявкам: создать, получить, обновить, список, статистика.

## Пользовательский сценарий

- `src/handlers/start.py` — `/start` и кнопка связи с админом.
- `src/handlers/user_orders.py` — оформление заявки, заполнение полей, просмотр своих заявок, оплата, отказ пользователя.
- `src/keyboards/user_keyboards.py` — кнопки пользователя: оформление, поля заявки, мои заявки, оплата, назад.
- `src/states/order_states.py` — FSM-состояния для ввода адреса, размера, ссылки, фото и комментария.

## Админский сценарий

- `src/handlers/admin_orders.py` — `/stats`, одобрение, ввод цены, отклонение по причинам, отмена админского действия.
- `src/keyboards/admin_keyboards.py` — кнопки админа: одобрить, отклонить, причины отказа, отменить действие.

## Бизнес-логика

- `src/services/order_service.py` — центральная логика заявок: обязательные поля, статусы, формат сообщений, цена, причина решения, время UTC+4.
- `src/services/payment_service.py` — место для подключения реального платёжного провайдера. Сейчас там заглушка.

## Что менять чаще всего

- Тексты сообщений заявки — `src/services/order_service.py`.
- Кнопки пользователя — `src/keyboards/user_keyboards.py`.
- Кнопки админа и причины отказа — `src/keyboards/admin_keyboards.py`.
- Логику отправки заявки админам — `src/handlers/user_orders.py`.
- Логику одобрения/отклонения — `src/handlers/admin_orders.py`.
- Поля заявки в БД — `src/models/order.py`, затем `src/database.py` для автодобавления колонок.

## Как устроен черновик

До отправки админам заявка не создаётся в PostgreSQL. Пользовательский черновик живёт во FSM. Это нужно, чтобы отменённый до отправки черновик не расходовал ID заявки и не оставлял мусор в базе.

Когда пользователь нажимает `Отправить заявку`, бот:

1. Проверяет обязательные поля.
2. Создаёт запись в PostgreSQL.
3. Получает реальный ID заявки.
4. Отправляет заявку в админ-чат.
5. Показывает пользователю карточку заявки уже с ID.

## Как устроено время

В базе время обработки хранится в UTC. В интерфейсе админов оно выводится как UTC+4, то есть на один час больше Москвы.
